# 보험사 금융 IT 인터페이스 통합관리시스템
## 최초 기획서 (v1.0)

---

# 1. 프로젝트 개요

보험사 내부 핵심 시스템(계약, 수납, 지급, 고객관리 등)과 외부 기관(은행, 제휴사, 공공기관 등) 간 연계되는 다양한 인터페이스를 통합 관리하는 운영 플랫폼을 구축한다.

운영 담당자가 여러 시스템을 개별적으로 확인하지 않고, 하나의 화면에서 상태 모니터링, 장애 대응, 로그 조회, 재실행까지 수행할 수 있도록 지원하는 것이 목표다.

---

# 2. 기획 배경 / 문제 정의

보험사 금융 시스템은 다수의 외부 기관과 실시간/배치 방식으로 연동된다.

예시:

- 보험료 납입 결과 은행 수신
- 보험금 지급 요청 데이터 송신
- 고객 정보 제휴사 연동
- 청구 데이터 공공기관 연계
- 정산 파일 배치 전송

기존 운영 환경에서는 다음 문제가 발생할 수 있다.

## 운영 상 문제점

- 인터페이스 상태 확인 화면이 분산되어 있음
- 장애 발생 시 탐지가 늦음
- 로그 조회 경로가 복잡함
- 수동 재처리 절차가 비효율적임
- 기관별 / 프로토콜별 관리 기준이 상이함

---

# 3. 프로젝트 목표

운영자가 단일 플랫폼에서 다음 업무를 빠르게 수행할 수 있도록 한다.

## 핵심 목표

- 인터페이스 전체 현황 확인
- 실패 건 즉시 탐지
- 에러 로그 신속 조회
- 재실행 및 복구 처리
- 기관별 상태 검색 및 필터링

---

# 4. 주요 사용자 정의

## Primary User : 인터페이스 운영 담당자

### 담당 업무

- 일일 인터페이스 정상 여부 점검
- 실패 건 확인 및 장애 대응
- 기관 문의 대응
- 처리 이력 확인
- 운영 현황 보고

---

# 5. 사용자 플로우

## Flow A. 일일 운영 점검

로그인  
→ 대시보드 접속  
→ 전체 성공률 / 실패 건수 확인  
→ 실패 건 존재 시 상세 화면 이동

---

## Flow B. 장애 대응 (핵심 플로우)

실패 상태 발견  
→ 인터페이스 상세 조회  
→ 최근 실행 로그 확인  
→ 에러 메시지 확인  
→ 재실행 버튼 클릭  
→ 성공 여부 확인

---

## Flow C. 기관별 조회

기관명 검색  
→ 국민은행 선택  
→ REST 프로토콜 필터 적용  
→ 최근 처리 상태 확인

---
